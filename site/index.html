Quickstart

Issue. Authorize. Delegate. Revoke.

One import. No config files. The passport carries the policy, the signature proves it hasn't been tampered with, and the authority tracks every action.

$ npm install @passport-agent/sdk
app.ts 
import { AgentPassport } from '@passport-agent/sdk'

// Human authorizes an agent
const passport = AgentPassport.issue({
  principal:   'user:alice@company.com',
  agent:       'agent:booking-bot',
  permissions: ['calendar:write'],
  limits:      { maxSpend: 500 },
})

// Check before every action
passport.authorize('calendar:write')
// ✓ allowed

passport.authorize('database:drop')
// ✗ PassportDeniedError

// Delegate to sub-agent (narrower only)
const child = passport.delegate({
  agent:       'agent:email-drafter',
  permissions: ['calendar:read'],
  limits:      { maxSpend: 0 },
})

// Kill the entire chain
passport.revoke()
Protocol

Four invariants, enforced cryptographically

I
Scoped delegation
Every passport is bound to a specific principal, agent, permission set, and time window. No ambient authority.
principal: user:alice@co.com
agent: agent:booking-bot
permissions: [calendar:write]
expires: 2025-01-16T00:00:00Z
signature: ed25519(...)
II
Monotonic narrowing
Child passports get a subset of the parent's permissions. Escalation is structurally impossible, enforced by the policy engine and Biscuit Datalog checks.
parent: [read, write, delete]
  ↳ child: [read, write]
    ↳ grandchild: [read]
      ↳ [write] DENIED
III
Budget enforcement
Per-passport spend tracking on the authority. Agents can't manipulate their own records. When the budget runs out, the passport stops working.
maxSpend: $500.00
spent: $347.20
remaining: $152.80
next request ($200): DENIED
IV
Cascade revocation
Revoke a parent and every descendant in the delegation tree is instantly killed. Persistent across server restarts.
revoke("passport-root")
✗ passport-root
  ✗ passport-child-1
  ✗ passport-child-2
    ✗ passport-grandchild

Security model

Every design choice favors safety over convenience

✓
Tamper-proof tokens
Ed25519 signatures verified on every authorize() call. Flip one bit and the passport is rejected.
✓
No privilege escalation
Policy engine and Biscuit Datalog checks enforce that child permissions are always a strict subset of parent.
✓
Replay protection
128-bit cryptographic nonce per passport combined with expiration timestamps.
✓
Server-side spend tracking
Budget enforcement on the authority, not the client. Agents can't lie about how much they've spent.
✓
Full audit trail
Every authorize() call logged with passport ID, action, result, and timestamp. Complete forensic history.
✓
Biscuit token support
Datalog policy blocks for deep delegation chains where intermediate agents may be untrusted.
