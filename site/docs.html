Documentation

Agent Passport gives AI agents cryptographically verifiable authorization tokens — scoped permissions, spend limits, delegation chains, and instant revocation.

💡
New here? Start with the Quick Start to issue your first passport in under 2 minutes.

Why Agent Passport?

AI agents are calling APIs, booking meetings, moving money, and managing infrastructure — but there's no standard way to scope what they're allowed to do, how much they can spend, or to revoke access instantly when something goes wrong.

Agent Passport sits between the identity layer and the action layer, answering one question: "Is this agent allowed to do this specific thing right now?"

  • Scoped permissions — colon-separated namespaces with wildcard matching
  • Spend limits — budget caps tracked server-side
  • Delegation chains — sub-agents get derived passports with narrowed scope
  • Cascade revocation — revoking a parent kills all descendants
  • Audit trail — every action logged with passport ID
  • Ed25519 signatures — cryptographic proof of authorization

Quick Start

Get up and running in three steps.

1. Install the SDK

bash 
npm install @passport-agent/sdk

2. Issue a passport

typescript 
import { PassportIssuer } from '@passport-agent/core'

const issuer = new PassportIssuer()

const passport = issuer.issue({
  principal: 'user:alice@company.com',
  agent:     'agent:booking-bot',
  permissions: ['calendar:read', 'calendar:write'],
  limits:    { maxSpend: 100, currency: 'USD' },
  ttl:       3600000, // 1 hour
})

3. Authorize an action

typescript 
const result = issuer.authorize(passport, 'calendar:write', 10)

if (result.allowed) {
  // proceed with the action
} else {
  console.log(`Denied: ${result.reason}`)
}
ℹ️
The PassportIssuer generates Ed25519 keys automatically. For production, you'll want to persist keys and run the authority server.

Passports

A passport is a signed token that encodes who authorized an agent, what it can do, how much it can spend, and when it expires.

Passport structure

json 
{
  "id":          "a1b2c3d4...",
  "iss":         "agent-passport:local",
  "sub":         "agent:booking-bot",
  "principal":   "user:alice@company.com",
  "permissions": [
    { "action": "calendar:read" },
    { "action": "calendar:write" }
  ],
  "limits": {
    "maxSpend": 100,
    "currency": "USD",
    "spent":    0
  },
  "iat":       1719475200000,
  "exp":       1719561600000,
  "parentId":  null,
  "nonce":     "e5f6a7b8..."
}
FieldTypeDescription
idstringUnique 128-bit hex identifier
issstringIssuing authority
substringAgent receiving the passport
principalstringHuman who authorized the delegation
permissionsPermission[]Allow-list of actions
limitsSpendLimitBudget constraints
iatnumberIssued-at timestamp (ms)
expnumberExpiration timestamp (ms)
parentIdstring | nullParent passport ID for delegation chains
noncestringReplay protection

Permissions

Permissions use colon-separated namespaces. The wildcard * matches any action within a namespace.

PermissionMatches
calendar:readExact: calendar:read only
calendar:*All calendar actions: calendar:read, calendar:write, etc.
*Everything (use with caution)
typescript 
import { matchesPermission } from '@passport-agent/core'

matchesPermission('calendar:read', [{ action: 'calendar:*' }])  // true
matchesPermission('admin:delete',  [{ action: 'calendar:*' }])  // false
matchesPermission('anything',      [{ action: '*' }])           // true

Delegation

An agent can delegate a subset of its own passport to a sub-agent. Privileges can only narrow, never escalate — this is called monotonic narrowing.

typescript 
// Parent has calendar:* and email:send
const parentPassport = issuer.issue({
  principal: 'user:alice@company.com',
  agent: 'agent:assistant',
  permissions: ['calendar:*', 'email:send'],
  limits: { maxSpend: 500 },
})

// Child gets only calendar:read, with $50 budget
const childPassport = issuer.delegate(parentPassport, {
  agent: 'agent:room-finder',
  permissions: ['calendar:read'],
  limits: { maxSpend: 50 },
})

Delegation rules

  • Permission subset — every child permission must be satisfiable by a parent permission
  • Spend cap — child maxSpend ≤ parent's remaining budget
  • Time bound — child exp ≤ parent exp
  • Principal preserved — the human principal is immutable across the chain
⚠️
Attempting to escalate privileges throws an error. You cannot grant a child permissions the parent doesn't have, or a budget exceeding the parent's remaining balance.

Revocation

Revoking a passport cascades to all descendants. If a parent is revoked, every child and grandchild in the chain is immediately invalid.

typescript 
// Revoke the parent — all children are also revoked
issuer.revoke(parentPassport.payload.id)

// This now fails even though childPassport wasn't directly revoked
issuer.authorize(childPassport, 'calendar:read', 0)
// → { allowed: false, reason: 'REVOKED' }

Revocation is immediate and irreversible. To restore access, issue a new passport.

Spend Limits

Every passport has a spend budget tracked server-side. Each authorize() call deducts from the budget. When the budget is exhausted, further spend actions are denied.

typescript 
const passport = issuer.issue({
  principal: 'user:alice@company.com',
  agent: 'agent:shopper',
  permissions: ['purchase'],
  limits: { maxSpend: 100, currency: 'USD' },
})

issuer.authorize(passport, 'purchase', 80)  // ✓ allowed ($80 spent)
issuer.authorize(passport, 'purchase', 30)  // ✗ denied (only $20 left)

Audit Trail

Every authorize() call produces an immutable audit entry — both allowed and denied actions are logged.

json 
{
  "id":         "audit-001",
  "passportId": "a1b2c3d4...",
  "action":     "calendar:write",
  "allowed":    true,
  "spend":      10,
  "timestamp":  1719475200000
}

MCP Plugin

Drop-in middleware for Model Context Protocol servers. Intercepts tool calls and checks passports before execution.

bash 
npm install @passport-agent/mcp
typescript 
import { withPassport } from '@passport-agent/mcp'

server.tool('book_meeting',
  withPassport('calendar:write', async (args) => {
    // Tool executes only if passport allows calendar:write
    return bookMeeting(args)
  })
)

HTTP Middleware

Extract and validate passports from HTTP headers. Works with Express, Fastify, and Next.js.

bash 
npm install @passport-agent/middleware
typescript 
import { passportGuard } from '@passport-agent/middleware'

app.use('/api', passportGuard())

app.post('/api/book', (req, res) => {
  // req.passport is verified and attached
  const { principal, permissions } = req.passport.payload
})

LangChain

Wrap LangChain tools with passport authorization. Supports both the generic ToolDefinition interface and the LangChain-compatible invoke pattern.

bash 
npm install @passport-agent/langchain
typescript 
import { wrapLangChainTool } from '@passport-agent/langchain'

const protectedTool = wrapLangChainTool(searchTool, passport, {
  issuer,
  permissionMapper: (name) => `tool:${name}`,
  callbacks: {
    onToolStart: (tool, input) => console.log(`Calling ${tool}`),
    onToolEnd:   (tool, output) => console.log(`Done: ${tool}`),
  },
})

const result = await protectedTool.invoke({ query: 'meetings' })

CrewAI

Build multi-agent pipelines where each agent gets a delegated passport with narrowed scope.

bash 
npm install @passport-agent/crewai
typescript 
import { createCrew, runPipeline } from '@passport-agent/crewai'

const crew = createCrew(
  {
    principal: 'user:alice@company.com',
    issuer,
    permissions: ['search', 'write', 'publish'],
    limits: { maxSpend: 500 },
  },
  [
    { name: 'researcher', role: 'Research', permissions: ['search'] },
    { name: 'writer',     role: 'Writer',   permissions: ['write'] },
  ],
)

const result = await runPipeline([
  { agent: crew[0], tasks: [{ description: 'Research topic', requiredPermission: 'search', execute: searchFn }] },
  { agent: crew[1], tasks: [{ description: 'Write report',  requiredPermission: 'write',  execute: writeFn }] },
], { stopOnFailure: true })

A2A Protocol

Extend Agent-to-Agent discovery with passport requirements. Agents can advertise required permissions in their Agent Card.

bash 
npm install @passport-agent/a2a
typescript 
import { createAgentCard, validatePassportForCard } from '@passport-agent/a2a'

const card = createAgentCard({
  name: 'Booking Agent',
  description: 'Books meetings and rooms',
  url: 'https://booking.example.com',
  capabilities: ['calendar', 'rooms'],
  passportRequirements: {
    requiredPermissions: ['calendar:write'],
    minSpendLimit: 50,
  },
})

const { valid, missingPermissions } = validatePassportForCard(passport, card)

PassportIssuer

The central class for issuing, validating, delegating, and revoking passports.

new PassportIssuer()

Creates a new issuer with a freshly generated Ed25519 key pair.

.issue(options)

issue(options: IssueOptions): SignedPassport
ParameterTypeDescription
principalstringHuman authorizing the agent
agentstringAgent identifier
permissionsstring[]List of permission strings
limits{ maxSpend, currency? }Optional spend constraints
ttlnumberTime-to-live in milliseconds

.authorize(passport, action, spend?)

authorize(passport: SignedPassport, action: string, spend?: number): AuthorizeResult

Checks if the passport allows the given action with the given spend amount. Returns { allowed: boolean, reason?: string }.

.delegate(parent, options)

delegate(parent: SignedPassport, options: DelegateOptions): SignedPassport

Creates a child passport with narrowed permissions. Throws if the child would escalate privileges.

.revoke(passportId)

revoke(passportId: string): void

Revokes a passport and all descendants in the delegation chain.

Types

typescript 
interface Permission {
  action: string
  resource?: string
}

interface SpendLimit {
  maxSpend: number
  currency: string
  spent: number
}

interface PassportPayload {
  id: string
  iss: string
  sub: string
  principal: string
  permissions: Permission[]
  limits: SpendLimit
  iat: number
  exp: number
  parentId: string | null
  nonce: string
}

interface SignedPassport {
  payload: PassportPayload
  signature: string
  publicKey: string
}

interface AuthorizeResult {
  allowed: boolean
  reason?: string
}

Crypto

Agent Passport uses Ed25519 for all signatures. Keys are generated automatically by PassportIssuer, but you can use the primitives directly.

typescript 
import { generateKeyPair, sign, verify, toHex } from '@passport-agent/core'

const { publicKey, privateKey } = generateKeyPair()

const data = 'hello world'
const sig = sign(privateKey, data)
const valid = verify(publicKey, data, sig) // true

Error Codes

When authorize() returns allowed: false, the reason field contains one of these codes:

CodeDescription
REVOKEDPassport or an ancestor has been revoked
EXPIREDCurrent time is past exp
PERMISSION_DENIEDNo matching permission for the requested action
SPEND_LIMIT_EXCEEDEDRequested spend exceeds remaining budget
INVALID_SIGNATURESignature verification failed

Security Model

  • Ed25519 signatures on every passport — tamper-proof
  • Server-side spend tracking — clients can't manipulate budgets
  • Nonces prevent replay attacks
  • Monotonic narrowing — delegation can only reduce privileges
  • Cascade revocation — one call kills the entire chain
  • Append-only audit log — every action is recorded
⚠️
Transmit passports over TLS only. Private keys must never leave the authority. Treat signed passports as bearer tokens.
